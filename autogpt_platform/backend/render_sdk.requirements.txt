# Single source of truth for the render_sdk pin (Render Workflows client).
#
# render_sdk is intentionally NOT in pyproject.toml: its transitive dep
# openapi-python-client (<0.27,>=0.26.1) pins ruff<0.14, which conflicts with
# this repo's ruff ^0.15, so Poetry cannot resolve it in the lockfile. It is
# installed --no-deps in backend/Dockerfile (only aiohttp/httpx are needed at
# runtime, and both are already backend deps). Hash-pinned here so the
# --no-deps install is reproducible and tamper-evident, and so the version
# lives in exactly one place. Both the wheel and sdist hashes are listed so
# pip can install from either under --require-hashes.
#
# The Dockerfile server stage and the manual Render Workflow service both
# install via `pip install --no-deps --require-hashes -r
# render_sdk.requirements.txt`, so neither restates the version — this file is
# the only place the pin appears.
